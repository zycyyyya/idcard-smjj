## 核心规则

### 规则1：唯一工具
**所有资料卡/销售卡制作，必须使用 `idcard-smjj` skill**

- ✅ 允许：使用 `idcard-smjj` skill 生成
- ❌ 禁止：使用旧版 `_gen_cards.js`、`_template_sales_card.js` 或其他脚本
- ❌ 禁止：手动创建 Word 文档（除非 skill 无法使用）

### 规则2：数据来源
1. **尽调报告**（桌面/私募/尽调报告合集/）- 第一优先级
2. **网络搜索验证** - 补充/验证信息
3. **私募排排网** - 业绩数据
4. **严禁编造** - 无来源数据必须标注"待确认"

### 规则3：输出标准
- **格式**：Word (.docx)
- **结构**：12标准板块 + 万神殿分析（4部分）
- **命名**：`机构名_销售卡片.docx`
- **路径**：`桌面/私募/私募销售工具卡/`

### 规则4：万神殿分析（强制包含）
第7部分必须包含完整4部分：
1. 10位大师对照表（5列）
2. 5条通用原则适配度
3. 管理人独特性（3-4条）
4. 大师语录对照（3-5条）

**万神殿数据来源（优先级）**：
1. **优先调用** `investment-masters` skill（`workspace/skills/investment-masters/`）
2. **Fallback** 使用内置大师库（硬编码）
3. 基于 AlphaGBM/investment-masters 项目数据

**执行逻辑**：
```
生成资料卡
    ↓
尝试调用 investment-masters skill
    ↓
成功 → 使用 skill 数据
失败 → 使用内置 MASTERS 数组
    ↓
生成万神殿分析
```

### 规则5：质量检查（生成前必须确认）
- [ ] 12板块完整
- [ ] 万神殿4部分完整
- [ ] FAQ有4个问题（非2-3个）
- [ ] 风险提示有4条（具体非笼统）
- [ ] 核心优势有3条（有数据支撑）
- [ ] 配色符合规范
- [ ] 页眉页脚正确

## Skill 使用方式（推荐 v1.1.0 新功能）

### 方式1：智能生成（推荐）
```bash
cd ~/.openclaw/workspace/skills/idcard-smjj

# 步骤1：生成数据模板
node skill.js --template 机构名_data.json

# 步骤2：填写数据后验证
node skill.js --validate 机构名_data.json

# 步骤3：使用智能万神殿模板生成
# 策略类型：债券/主观/价值/量化/CTA/医药
node skill.js --data 机构名_data.json --output 机构名_销售卡片.docx --strategy 债券
```

### 方式2：基础生成
```bash
node skill.js --data 机构数据.json --output 销售卡片.docx
```

### 方式3：Node.js模块
```javascript
const { generateSalesCard } = require('./index.js');
await generateSalesCard(data, '输出路径.docx');
```

## 最佳实践（v1.1.0）

### 快速制作流程
```
1. 确认机构策略类型（债券/主观/价值/量化/CTA/医药）
        ↓
2. 生成数据模板：node skill.js --template 机构_data.json
        ↓
3. 填写基础信息（从尽调报告/网络搜索获取）
        ↓
4. 验证数据：node skill.js --validate 机构_data.json
        ↓
5. 智能生成：node skill.js --data 机构_data.json --output 机构_销售卡片.docx --strategy [类型]
        ↓
6. 检查质量报告，确认无误后交付
```

### 策略类型选择指南
| 机构类型 | 策略参数 | 说明 |
|---------|---------|------|
| 债券策略私募 | `--strategy 债券` | 滴海、富涌谷等 |
| 主观多头私募 | `--strategy 主观` | 枫泉投资等 |
| 价值投资私募 | `--strategy 价值` | 高恩私募等 |
| 量化私募 | `--strategy 量化` | 指增、市场中性 |
| CTA私募 | `--strategy CTA` | 管理期货策略 |
| 医药主题私募 | `--strategy 医药` | 专注医药投资 |

### 常见问题
**Q：智能模板生成的万神殿对照表需要修改吗？**  
A：建议根据机构实际情况微调，但智能模板已提供合理的基础框架。

**Q：数据验证不通过怎么办？**  
A：根据错误提示补充缺失字段，警告项建议完善但非强制。

**Q：没有尽调报告怎么办？**  
A：通过网络搜索（私募排排网、公司官网、新闻报道）收集信息，标注数据来源。
