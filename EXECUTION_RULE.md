# IDcard-smjj 执行规范（强制性）

## 生效日期
2026-05-08 起生效

## 规范级别
**强制性** - 所有私募基金管理人销售卡片制作必须遵守

---

## 核心规则

### 规则1：唯一工具
**所有资料卡/销售卡制作，必须使用 `idcard-smjj` skill**

- ✅ 允许：使用 `idcard-smjj` skill 生成
- ❌ 禁止：使用旧版 `_gen_cards.js`、`_template_sales_card.js` 或其他脚本
- ❌ 禁止：手动创建 Word 文档（除非 skill 无法使用）

### 规则2：数据来源
1. **尽调报告**（桌面/私募/尽调报告合集/）- 第一优先级
2. **网络搜索验证** - 补充/验证信息
3. **私募排排网** - 业绩数据
4. **严禁编造** - 无来源数据必须标注"待确认"

### 规则3：输出标准
- **格式**：Word (.docx)
- **结构**：12标准板块 + 万神殿分析（4部分）
- **命名**：`机构名_销售卡片.docx`
- **路径**：`桌面/私募/私募销售工具卡/`

### 规则4：万神殿分析（强制包含）
第7部分必须包含完整4部分：
1. 10位大师对照表（5列）
2. 5条通用原则适配度
3. 管理人独特性（3-4条）
4. 大师语录对照（3-5条）

**万神殿数据来源（优先级）**：
1. **优先调用** `investment-masters` skill（`workspace/skills/investment-masters/`）
2. **Fallback** 使用内置大师库（硬编码）
3. 基于 AlphaGBM/investment-masters 项目数据

**执行逻辑**：
```
生成资料卡
    ↓
尝试调用 investment-masters skill
    ↓
成功 → 使用 skill 数据
失败 → 使用内置 MASTERS 数组
    ↓
生成万神殿分析
```

### 规则5：质量检查（生成前必须确认）
- [ ] 12板块完整
- [ ] 万神殿4部分完整
- [ ] FAQ有4个问题（非2-3个）
- [ ] 风险提示有4条（具体非笼统）
- [ ] 核心优势有3条（有数据支撑）
- [ ] 配色符合规范
- [ ] 页眉页脚正确

---

## 执行流程

```
用户请求制作资料卡
        ↓
确认机构名称
        ↓
查找尽调报告（桌面/私募/尽调报告合集/）
        ↓
提取PDF内容 → 网络搜索验证补充
        ↓
按12板块整理数据（JSON格式）
        ↓
使用 idcard-smjj skill 生成
        ↓
质量检查（对照上述清单）
        ↓
交付：机构名_销售卡片.docx
```

## Skill 使用方式

### 方式1：CLI命令
```bash
cd ~/.openclaw/workspace/skills/idcard-smjj
node skill.js --data 机构数据.json --output 销售卡片.docx
```

### 方式2：Node.js模块
```javascript
const { generateSalesCard } = require('./index.js');
await generateSalesCard(data, '输出路径.docx');
```
